РУКОВОДСТВО ПО ОБНОВЛЕНИЮ SITEMAP
═══════════════════════════════════════════════════════════════

1. ПРИОРИТЕТЫ (priority):
   - 1.0: Главная страница
   - 0.9: Основные разделы (О проекте)
   - 0.8: Контакты, важные landing pages
   - 0.7: Категории игры, популярные страницы
   - 0.6: Правовые документы, страницы 18+
   - 0.3-0.5: API, служебные страницы
   - 0.1-0.2: Технические страницы, 404

2. ЧАСТОТА ОБНОВЛЕНИЙ (changefreq):
   - daily: Главная (если контент меняется ежедневно)
   - weekly: Игровые страницы, часто обновляемый контент
   - monthly: Основные разделы, О проекте
   - yearly: Правовые документы, контакты

3. LASTMOD (дата последнего изменения):
   - Используйте ISO 8601 формат: YYYY-MM-DDTHH:MM:SS+TZ
   - Обновляйте при каждом изменении контента
   - Автоматизируйте обновление в deploy скриптах

4. МОБИЛЬНОСТЬ:
   - Добавляйте <mobile:mobile/> для всех страниц, 
     оптимизированных для мобильных устройств

5. ИЗОБРАЖЕНИЯ:
   - Добавляйте важные изображения для лучшей индексации
   - Используйте описательные title и caption
   - Указывайте полные URL

6. АВТОМАТИЗАЦИЯ:
   - Создайте скрипт для автогенерации sitemap
   - Обновляйте при добавлении новых страниц
   - Уведомляйте поисковики: 
     http://www.google.com/ping?sitemap=URL
     http://www.bing.com/ping?sitemap=URL

7. ВАЛИДАЦИЯ:
   - Google Search Console > Sitemaps
   - https://www.xml-sitemaps.com/validate-xml-sitemap.html
   - Проверьте robots.txt на ссылку на sitemap

8. ДОПОЛНИТЕЛЬНЫЕ SITEMAP:
   - sitemap-images.xml (только изображения)
   - sitemap-news.xml (новости, если будут)
   - sitemap-video.xml (видео контент)

9. РАЗМЕР И ОГРАНИЧЕНИЯ:
   - Максимум 50,000 URL в одном файле
   - Максимальный размер: 50MB
   - При превышении создавайте sitemap index

10. МОНИТОРИНГ:
    - Отслеживайте ошибки в Search Console
    - Проверяйте покрытие индексации
    - Анализируйте статистику сканирования

ПОМНИТЕ: Этот sitemap нужно обновлять при каждом 
изменении структуры сайта или добавлении нового контента!
